# PROJECT REQUIREMENTS:

The program contains three kinds
of objects: Staff, Student, and Faculty. For each object, the program stores relevant information such as university ID, name,
etc. Different information is stored depending on the type of object. For example, a student has a gpa, a faculty has a title
and department (professor, mathematics).

- Implement a inheritance hierarchy that allows Students, Faculty, and Staff to inherit traits from relevant classes as necessary.
- Create functions to add, remove, modify, and otherwise manipulate all pieces of information about a student/staff member.
- Each member on the list should have a unique identifier assigned to them at creation, that can be used to reference them within the database.
- Create a report that lists all Faculty, Staff, and Students in a .txt file at the conclusion of the program's runtime.

Store information such as:
- Student GPA
- Faculty/Staff ID
- Faculty Department
- etc.

This information should be displayed and formatted correctly, with students sorted by GPA.
